Integration with Context Builder
=================================

The ISOLATE step fits into the pipeline as:

SELECT -> ORDER -> COMPRESS -> ISOLATE -> SANITIZE
                                  |
                    (New step integrated here)

The ContextIsolator evaluates candidate changes from the context
and determines whether the Agent execution should be isolated to:
- No isolation (NONE): For safe read-only operations
- Worktree isolation (WORKTREE): For code/config changes
- Container isolation (CONTAINER): For risky infrastructure/DB changes

This feeds into downstream orchestration to launch the appropriate
execution environment.

Usage Pattern
=============

from isolators.context_isolator import ContextIsolator

isolator = ContextIsolator()

result = await isolator.determine_isolation(
    context={
        'candidates': [
            {'file_path': 'src/main.py', 'content_type': 'code', 'operation': 'modify'}
        ]
    },
    agent_id='A1'
)

# result['isolation_mode'] -> 'WORKTREE'
# result['risk_level'] -> 'low'
# result['duration_ms'] -> 0.09
