===============

    ├─ PostgresSource

    │   ├─ _query_requirement_history()
    │   ├─ _query_related_prs()
    │   └─ _query_existing_specs()
    │
    ├─ Neo4jSource
    │   ├─ _get_driver()

DATA FLOW SEQUENCE
==================

SELECT STAGE EXECUTION:

1. REQUEST
   - target_agent: str (e.g., "A9")
   - req_id: str
   - query_text: str (optional)
   - max_tokens: int

2. PARALLEL QUERIES

   PostgresSource.query(req_id)
   - _query_requirement_history()
   - _query_related_prs()
   - _query_existing_specs()
   Output: [candidate, candidate, ...]

   VectorSource.query(req_id, query_text)
   - _query_similar_requirements()
   - _query_historical_knowledge()
   - _query_design_patterns()
   Output: [candidate, candidate, ...]

   Neo4jSource.query(req_id) [if configured]
   - _query_upstream_dependencies()
   - _query_downstream_dependencies()
   - _query_service_calls()
   Output: [candidate, candidate, ...]

3. CANDIDATE STRUCTURE
   {
       'source': 'postgres_requirements' | 'vector_similar_reqs' | 'neo4j_upstream_deps'
       'source_id': str
       'req_id': str
       'type': 'spec' | 'code' | 'knowledge' | 'doc'
       'title': str
       'content': str
       'metadata': Dict
       'relevance': float (0.0-1.0)
       'tags': Optional[List[str]]
   }

4. MERGE & DEDUPLICATE
   - Group by (source_id, source)
   - Keep max(relevance) when duplicates found
   - Merge metadata
   - Tag duplicates: 'dup_from_{source}'
   - Sort by relevance DESC
   Output: [merged_candidate, ...]

5. BUILD CONTEXT ITEMS
   For each candidate:
   - Estimate tokens (~3 chars/token)
   - Check against max_tokens budget
   - Create ContextItem with:
     * type, content, relevance, position='mid'
     * tokens (estimated), source, source_id, tags
   - Update tokens_remaining OR increment discarded
   Output: ContextItem[], tokens_used, discarded

6. METRICS RECORDING
   - record_candidates(source, count)
   - record_deduped(count)
   - record_final(count)
   - record_duration(ms)

7. RETURN SelectResult
   {
       'items': [ContextItem, ...],
       'tokens_used': int,
       'discarded': int
   }

8. NEXT STAGE (ORDER)
   ContextOrderer.order(items, max_tokens)


CANDIDATE ORIGIN MAPPING
========================

PostgreSQL Source:

  postgres_requirements
  - Input: req_id
  - Output: requirement history (versions, status, metadata)
  - Relevance: 0.80-0.85

  postgres_prs
  - Input: req_id
  - Output: related PRs (merged, in-review, draft)
  - Relevance: 0.75-0.80

  postgres_specs
  - Input: req_id
  - Output: existing specifications
  - Relevance: 0.80-0.85

Neo4j Source:

  neo4j_upstream_deps
  - Input: req_id (service)
  - Output: upstream service dependencies (3-hop)
  - Relevance: 0.65-0.72

  neo4j_downstream_deps
  - Input: req_id (service)
  - Output: downstream service dependencies (3-hop)
  - Relevance: 0.60-0.68

  neo4j_service_calls
  - Input: req_id (service)
  - Output: direct service-to-service calls
  - Relevance: 0.70-0.75

Vector Source:

  vector_similar_reqs
  - Input: req_id + query_text
  - Output: semantically similar requirements (>0.5 similarity)
  - Relevance: 0.60-0.75

  vector_historical_knowledge
  - Input: req_id + query_text
  - Output: design decisions, architectural patterns (>0.45 similarity)
  - Relevance: 0.50-0.70

  vector_design_patterns
  - Input: req_id + query_text
  - Output: related design patterns and solutions (>0.5 similarity)
  - Relevance: 0.60-0.75


CONNECTION MANAGEMENT
=====================

PostgreSQL Connection Pool:
  - Created: async pool with min_size=1, max_size=5
  - Lifecycle: Per MultiSourceSelector instance
  - Cleanup: await selector.close()
  - Query timeout: Default asyncpg (30s)

Neo4j Connection:
  - Driver: Lazy-initialized on first query
  - Session: Created per query (automatic cleanup)
  - Fallback: Returns empty list if driver unavailable
  - Cleanup: await selector.close() calls driver.close()

Embedder (singleton):
  - Instance: get_embedder() returns singleton
  - Fallback: Hash-based simulation if model unavailable
  - Degradation: Graceful fallback on embedding errors


ERROR RECOVERY PATHS
====================

PostgreSQL Query Failure:
  try:
      rows = await conn.fetch(query)
  except:
      logger.error(...)
      return []  # Fallback: empty list

  Impact: Other sources continue, no cascade failure

Neo4j Query Failure:
  if driver is None:
      logger.warning(...)
      return []  # Skipped gracefully

  Impact: Falls back to PostgreSQL + Vector sources

Vector Embedding Failure:
  try:
      embedding = embedder.embed(text)
  except:
      logger.error(...)
      return []  # Fallback: empty list

  Impact: Vector source disabled, others continue

Token Budget Exceeded:
  if estimated_tokens > tokens_remaining:
      discarded += 1  # Count overflow
      continue  # Skip this candidate

  Impact: Item excluded from result, tracked as discarded

Deduplication Conflicts:
  if candidate_relevance > existing_relevance:
      merged[key]['relevance'] = candidate_relevance
      merge metadata  # Keep best version

  Impact: Single best candidate per (source_id, source)
