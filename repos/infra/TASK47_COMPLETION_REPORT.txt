KEY IMPLEMENTATION DETAILS

Alert Rules (15 total):

- LLM monitoring: HighLLMLatency

- Pipeline: HighWIP, LoopTripped, GateSLAOverdue, ZeroThroughput
- System: HighEventBusLatency, SlowContextBuilder, WebSocketConnectionDrop
- Quality: TestFailureSpike, DatabaseConnectionPoolExhausted

Feishu Message Format:
- Title: Emoji + Alert name (e.g., "🔴 告警触发 - NATSConnectionDown")
- Header: Color-coded by severity (red/orange/blue/green)
- Body: Description + Component + Severity + Timestamp
- Actions: Buttons to Grafana and Prometheus dashboards

Alert Lifecycle:
1. Prometheus evaluates rules every 15 seconds
2. Alert fires when condition met for specified duration (2-5 min)
3. AlertManager groups and routes via webhook
4. mc-backend formats and sends to Feishu
5. Alert resolves when condition clears
6. Resolution notification sent to Feishu
