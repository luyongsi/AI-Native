================================================================================
VALIDATION & RETRY LOGIC
================================================================================

Process:
1. Generate spec via LLM
2. Parse JSON response
3. Validate against OpenAPI 3.1
4. If invalid: attempt auto-fix (up to 3 times)
5. Log all attempts and fixes
6. Return result with validation status

Automatic Fixes:
- Add missing "openapi": "3.1.0"
- Add missing info object
- Add missing paths with health endpoint
- Ensure all required fields present
