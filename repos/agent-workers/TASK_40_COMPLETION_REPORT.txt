========================================
ARCHITECTURE & DATA FLOW
========================================

Request Flow:
┌──────────────────┐
│ Frontend Canvas  │ User draws annotations on prototype image
└────────┬─────────┘
         │ HTTP POST /api/prototypes/annotate
         ▼
┌──────────────────────────────────────┐
│ Backend API (prototypes.py)          │
│ - Validate annotation data           │
│ - Store to PostgreSQL                │
│ - Publish NATS event                 │
└────────┬─────────────────────────────┘
         │ NATS: prototype.annotated.{req_id}
         ▼
┌──────────────────────────────────────┐
│ A3 Agent Worker (a3_ui_generator.py) │
│ - Parse annotations to requirements  │
│ - Call LLM to generate code          │
│ - Publish artifact event             │
└────────┬─────────────────────────────┘
         │ NATS: agent.A3.artifact
         ▼
┌──────────────────────────────────────┐
│ Frontend Activity Stream             │
│ - Receive artifact event via SSE     │
│ - Display code in CodePreview        │
└──────────────────────────────────────┘

Data Schemas:

Annotation:
{
  id: string,
  type: "component" | "interaction" | "data-binding",
  x: number, y: number, width: number, height: number,
  label: string,
  properties: Record<string, any>
}

UI Requirements (from A3 parser):
{
  components: [{id, name, position, size, properties}],
  interactions: [any],
  data_bindings: [any]
}

Activity Event:
{
  agent_id: "A3",
  req_id: string,
  event_type: "agent.A3.artifact",
  artifact_type: "ui_code_patch",
  artifact: {code: string, type: "tsx", ...}
}
