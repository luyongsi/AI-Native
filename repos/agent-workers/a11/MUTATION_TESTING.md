# A11 Mutation Testing Engine + Critic Mode

Comprehensive mutation testing integration for the A11 Auto Test Agent, combining mutmut (Python) and Stryker (JavaScript/TypeScript) with LLM-powered test generation via Critic mode.

## Overview

The mutation testing engine provides:

1. **Unified Mutation Testing Interface** — Execute mutmut or Stryker depending on project language
2. **Survived Mutation Analysis** — Parse and extract details about mutations that escaped testing
3. **A11 Critic Mode** — LLM-powered generation of targeted tests to kill survived mutations
4. **Test File Generation** — Write generated tests to project test directories
5. **Prometheus Metrics** — Track mutation scores, improvements, and Critic mode effectiveness
6. **Automatic Retry** — Re-run mutation tests after test augmentation to verify improvement

## Architecture

### Components

#### 1. MutationTester (`mutation_tester.py`)
Executes mutation tests using mutmut (Python) or Stryker (JS/TS).

```python
from a11.mutation_tester import MutationTester

tester = MutationTester(timeout=300)

# Python projects
result = await tester.run_mutmut("/path/to/project", target_file="src/utils.py")

# JavaScript/TypeScript projects
result = await tester.run_stryker("/path/to/project", config_path="stryker.conf.json")

# Returns:
# {
#     "survived": [{"id": "...", "mutator": "...", "location": {...}, ...}],
#     "killed": [...],
#     "mutation_score": 0.85,
#     "total_mutations": 20
# }
```

**Key Features:**
- Async subprocess execution with timeout protection
- JSON parsing for both mutmut and Stryker outputs
- Fallback stdout parsing if JSON reports unavailable
- Comprehensive error handling and logging

#### 2. CriticMode (`critic_mode.py`)
Analyzes survived mutations and generates targeted test cases using LLM.

```python
from a11.critic_mode import CriticMode

critic = CriticMode(llm_client=claude_client)  # Optional LLM

# Generate tests for survived mutations
tests = await critic.analyze_and_generate(
    survived_mutations=[...],
    source_code="def add(a, b): return a + b",
    language="python",
    max_tests=10
)

# Returns list of test cases with generated test code
```

**Key Features:**
- Mutation-aware prompt engineering
- Code context extraction around mutation locations
- Supports Python, JavaScript, and TypeScript
- Mock generation fallback when LLM unavailable
- Test code extraction from LLM responses

#### 3. TestFileWriter (`test_file_writer.py`)
Writes generated tests to project test directories.

```python
from a11.test_file_writer import TestFileWriter

writer = TestFileWriter()

file_path = writer.write_tests(
    tests=generated_tests,
    project_path="/path/to/project",
    language="python",
    output_file="test_critic.py"
)
```

**Key Features:**
- Language-aware test file generation
- Proper formatting and header comments
- Timestamp-based filenames
- Directory creation if needed

#### 4. MutationMetrics (`mutation_metrics.py`)
Prometheus metrics for mutation testing and Critic mode.

```python
from a11.mutation_metrics import MutationMetrics

metrics = MutationMetrics()

metrics.record_mutation_result(
    project="my_project",
    language="python",
    mutation_score=0.85,
    survived=3,
    killed=17,
    total=20
)

metrics.record_critic_mode_triggered("my_project", "python")
metrics.record_critic_tests_generated("my_project", "python", count=5)
metrics.record_critic_improvement(
    "my_project", "python",
    improvement=0.1,  # 10% improvement
    execution_time_ms=8500
)
```

**Available Metrics:**
- `a11_mutation_score` — Current mutation score (0-1)
- `a11_survived_mutations_count` — Number of survived mutations
- `a11_killed_mutations_count` — Number of killed mutations
- `a11_total_mutations_count` — Total mutations generated
- `a11_critic_tests_generated_total` — Tests generated by Critic mode (counter)
- `a11_critic_mode_triggered_total` — Critic mode trigger events (counter)
- `a11_mutation_score_improvement` — Score improvement histogram
- `a11_critic_execution_time_ms` — Critic execution time histogram

## Integration with A11 Agent

The mutation testing engine is integrated into `a11_auto_test_agent.py`:

```
Phase 5: Run mutation testing
├─ Execute mutmut (Python) or Stryker (JS/TS)
├─ Parse results and record metrics
└─ Check if score < 0.80

Phase 5.5: A11 Critic Mode (if triggered)
├─ Analyze survived mutations
├─ Generate targeted test cases via LLM
├─ Write tests to test file
├─ Re-run mutation testing
└─ Record improvement metrics

Phase 6-8: Coverage measurement and reporting
```

### Trigger Conditions

Critic mode is automatically triggered when:
- Initial mutation score < 0.80 (80%)
- Survived mutations detected

### Output

The final test report includes:
```json
{
  "mutation_score": 0.85,
  "mutation_result": {
    "score": 0.85,
    "survived": 3,
    "killed": 17,
    "total": 20
  },
  "critic_mode_triggered": true,
  "tests_augmented": 5
}
```

## Configuration

### mutmut (Python)

Create `.mutmut-config` in project root:

```ini
[mutmut]
paths_to_mutate=src/
tests_dir=tests/
runner=python -m pytest
timeout=10
```

### Stryker (JavaScript/TypeScript)

Create `stryker.conf.json` in project root:

```json
{
  "mutator": "javascript",
  "packageManager": "npm",
  "testRunner": "jest",
  "coverageAnalysis": "perTest",
  "thresholds": {
    "high": 80,
    "low": 60,
    "break": 50
  },
  "mutate": [
    "src/**/*.js",
    "!src/**/*.test.js"
  ]
}
```

## LLM Integration

The Critic mode supports any LLM client with an async `generate()` method:

```python
class MyLLMClient:
    async def generate(self, prompt: str, temperature: float = 0.2) -> str:
        # Your LLM implementation
        return generated_text

critic = CriticMode(llm_client=MyLLMClient())
```

If no LLM client is provided, Critic mode uses mock test generation.

## Test Generation Quality

Generated tests target specific mutations by:

1. **Extracting code context** around the mutation location
2. **Building mutation-aware prompts** that explain what the mutation does
3. **Requesting minimal, focused tests** that specifically exercise the mutated behavior
4. **Enforcing proper formatting** for the target language

Example prompt:
```
Mutation Type: ArithmeticOperator
Location: Line 15
Original: x + y
Mutated: x - y

Generate a test that FAILS with the mutated code but PASSES with the original.
```

## Troubleshooting

### mutmut not found

Install with: `pip install mutmut`

### Stryker timeout

Increase timeout in MutationTester:
```python
tester = MutationTester(timeout=600)  # 10 minutes
```

### No mutation results

Check that:
1. Source files are in configured paths
2. Test files exist and run successfully
3. mutmut/Stryker can access the project directory
4. Configuration files are valid

### Critic mode generating empty tests

Ensure:
1. Source code is readable and not empty
2. Survived mutations list is not empty
3. LLM client (if provided) is working
4. Test extraction patterns match generated code format

## Performance Considerations

- **Mutation testing** is CPU-intensive; allocate sufficient resources
- **LLM calls** add latency; consider batching mutations by type
- **Test re-runs** after Critic mode may double execution time
- Set appropriate timeouts based on project size and test suite speed
