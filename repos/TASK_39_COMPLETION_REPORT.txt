## Technical Implementation Details

### Neo4j Connection
- **URI**: `neo4j://172.27.78.109:7687` (configurable)
- **Auth**: `neo4j` / `ai-native-2026`
- **Driver**: AsyncGraphDatabase (async/await compatible)

### K14: DependencyTopology Process
1. Create Requirement node with metadata
2. Parse API schema paths and methods
3. Create APIEndpoint nodes
4. Create DEFINES relationships (Req → API)
5. Parse ERD entities
6. Create Database nodes
7. Create CREATES relationships (Req → DB)
8. Extract foreign key relationships
9. Create DEPENDS_ON edges between tables

### K15: ChangePropagation Process
1. Locate changed entity in graph
2. Traverse DEPENDS_ON/DEFINES/CREATES/QUERIES/CALLS relationships
3. Collect unique affected nodes
4. Calculate risk level based on affected count
5. Generate risk factors and recommendations
6. Optional: Trace reverse (upstream) dependencies

### Dual-Phase A12 Analysis
**Phase 2 (Pattern)**:
- Fast file path pattern matching
- Deterministic results
- No external dependencies

**Phase 3 (Graph)**:
- Neo4j dependency queries
- Accurate entity relationships
- Optional (graceful fallback)

**Merging**:
- Take maximum risk level from both phases
- Aggregate affected modules
- Combine recommendations
- Add graph-specific insights

### Error Handling Strategy
- **Neo4j unavailable**: Graceful degradation (K14/K15 optional)
- **Entity not found**: Return empty results with "not found" error
- **Query failures**: Log and return error details
- **Connection timeouts**: Retry with backoff (not implemented, could be added)

## Performance Considerations

- Neo4j indices on frequently queried fields

### Scalability
- Async/await for concurrent operations
- Batch operations supported
- Horizontal scaling: Neo4j can run in cluster mode

### Potential Bottlenecks
- Very deep graphs (depth > 5): Consider caching
