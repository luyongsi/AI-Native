## 二、一页纸全景

```
需求(多源汇入)
   │  P1: ①Requirement Intake  ②Knowledge Analyst        ──▶ Gate0 需求确认 ★
   ▼
设计  P2: ③UI Generator ④Spec Writer ⑤Design Review Panel ──▶ Gate1 设计确认 ★
   ▼
拆解  P3: ⑥Spec Decomposer ⑦Test Case Gen ⑧Architecture Expert ─▶ Gate2 架构确认 ★
   ▼
编码测试 P4: ⑨Dev Agent(双脑) ⑩CI/CD ⑪Auto Test(双脑·变异) ⑫Code Review
   │            └─ 内部微循环≤2 ┘   └── 外部大循环≤3 ──┘            ──▶ Gate3 发布确认 ★
   ▼
交付  P5: ⑬Release Agent(金丝雀+监控+自动回滚)
   ▲
全局守护(旁路): ⑭Knowledge Keeper  ⑮Change Propagation
人机界面: 🖥️ Mission Control 指挥舱（全程可观测 + 在 ★ 处审批）
控制面大脑: ⓪ Orchestrator（状态机 + DAG + 熔断 + Gate + 上下文构建）
```

**三句话讲清这套系统的与众不同**：
1. **环境即裁判**——质量不靠 LLM 互相点头，靠编译/测试/变异测试/扫描等客观信号说话。
2. **双脑自审 + 多级熔断**——Agent 内部"生成者 vs 环境锚定审查者"先自我收敛；不收敛就清上下文、升级、最终交人。
3. **人在四道门掌舵**——AI 干 ~80% 的确定性活，人在需求/设计/架构/发布四个关键点拍板，全程在指挥舱可见。

## 三、统一术语表（三份分册共用）

| 术语 | 定义 | 详见 |
|---|---|---|
| **环境即裁判 / 环境锚定 Critic** | 一切质量判定以编译、类型、Lint、单测/集成/E2E、变异测试得分、安全扫描等**客观信号**为准；LLM 负责生成与失败解释，不给自己打通过分 | 01 §1.2、02 §5 |
| **双脑（Generator–Critic）** | Agent 内部拆为"生成者"与"环境锚定审查者"，Critic 只看产物 + 信号、看不到生成者思考过程（上下文隔离） | 02 §5 |
| **内 / 外 / 辩论循环** | 内部微循环 ≤2 次、外部大循环 ≤3 次、设计辩论 ≤3 轮；超限即熔断降级 | 02 §6 |
| **上下文污染清理** | 外部循环连续失败后强制清空对话历史，重注入原始 Spec + 最新失败日志，打断"自条件化错误" | 02 §2.6 / §6.3 |
| **Gate 0–3** | 需求 / 设计 / 架构 / 发布四道人工门禁（HITL），各有 SLA 与超时升级 | 02 §7、03 §5 |
| **三层架构 + 指挥舱** | Control Plane（控制面）· Agent Matrix（执行面）· Global Sidecar（全局守护）+ Mission Control（指挥舱） | 02 §1 |
| **Orchestrator（⓪）** | 控制面编排大脑：状态机推进、DAG 调度、熔断、Gate 拦截、上下文构建 | 02 §2.1 |
| **15 个 Agent** | ①需求 ②知识 ③UI ④Spec ⑤设计评审 ⑥拆解 ⑦测试生成 ⑧架构 ⑨开发 ⑩CI/CD ⑪自动测试 ⑫代码审查 ⑬发布 ⑭知识沉淀 ⑮变更传播 | 02 §4 |
| **AITP** | AI Task Protocol，Worker（Claude Code/Cursor/Codex…）接入控制面的六动作协议 | 02 §3 |
| **AI 贡献度 / 节省比例等 KPI** | 配套原型中的**示例/目标值**，落地后须用真实埋点替换，不作对外承诺值 | 01 §7、03 §3.7 |

## 四、关键数据速查（出处见 01 参考文献）

| 主题 | 数据 | 用途 |
|---|---|---|
| 模型能力 | SWE-bench Verified ≈88%，但抗污染的 SWE-bench Pro 仅 ≈59% | 论证"够强但不稳，需编排兜底" |
| 多 Agent | 比单 Agent +90.2%，但 token ≈15×；token 解释 ~80% 性能方差 | 论证"高价值环节用多 Agent + 成本治理" |
| 测试质量 | Meta TestGen-LLM：75% 编译/57% 稳定通过/25% 提覆盖，73% 被采纳 | 论证变异测试 Critic 工业可行 |
| 生产力两面 | Copilot +26% 吞吐 / 55% 提速；METR 资深者反慢 19% | 论证"裸用 AI 不够，需系统" |
| 上下文 | Context Rot：18/18 模型随长度退化；中段降幅 >30% | 论证上下文隔离 + 污染清理 |

## 五、本版本相对原始草稿的主要改进

1. **补齐数据支撑**：把"看趋势/看竞争/看痛点"的论断逐条挂上可核查的外部研究与基准数据（SWE-bench、Anthropic、Meta、METR、Chroma 等），并区分"外部引用值"与"设计目标值"。
2. **修正逻辑自洽性**：
   - 将"抛弃 LLM 互评"与"双脑互审"的矛盾收敛为**环境锚定 Critic**；
   - 把 Agent 编号与端到端流程图对齐（含此前遗漏的 Knowledge Analyst）；
   - 统一为 **Gate 0–3** 四道门，并给出与已落地前端（Gate 1/2/3）的映射。
3. **覆盖端到端目标**：以多 Agent 编排为主线，向上接"需求多源汇入与原型"，向下接"发布与监控"，并把面向人的 Mission Control 作为协同界面纳入，真正覆盖"需求→发布大部分由 AI 实现"。
4. **分读者成册**：决策、工程、产品三视角各自成文又交叉引用，便于不同群体各取所需。
