# A1 需求分析 Agent 长期演进设计 v2.0

## 背景与现状

### 当前v1.0限制
- **单次交互**：无法澄清模糊点，用户输入"开发订单系统"时无法追问具体子模块
- **孤立分析**：不使用历史需求，每次分析都是零基础开始
- **固化流程**：单一LLM调用 + 关键词fallback，无动态策略调整
- **有限智能**：仅依赖通用LLM（DeepSeek），无领域专用增强

### v3.0演进愿景
- **多轮对话主动澄清**：置信度<70%时主动生成澄清问题
- **知识库增强分析**：接入A2历史需求库，提供相似案例和技术栈推荐
- **Skills插件扩展**：可插拔的风险评估、成本估算、领域检测等增强模块
- **自适应策略**：根据需求复杂度动态调整分析深度和调用链路

## 核心架构

### 1. 持续对话管理器（Continuous Dialogue Manager）

#### 核心理念
A1不是"问答式"澄清（问3个问题就结束），而是**持续协作式**需求完善：
- 人工可随时补充信息，无轮次限制
- A1持续监听需求更新，实时反馈理解
- 完成标志是人工明确确认"需求已完善"，而非固定轮次

#### 状态机
```
DRAFT (初始需求)
  ↓
ACTIVE (持续对话中) ←──────┐
  ↓                        │
  • A1分析当前需求          │
  • 生成理解+待澄清点        │
  • 人工review并补充 ────────┘
  ↓
CONFIRMED (人工确认完善)
  ↓
COMPLETED (进入下游流程)
```

**关键差异**：
- 旧模式：A1主动问 → 用户被动答 → 固定3轮
- 新模式：A1持续理解 → 人工主动补充 → 无轮次限制

#### 数据结构
```python
class ContinuousDialogueState(BaseModel):
    status: str = "active"           # draft | active | confirmed | completed
    iterations: int = 0              # 迭代次数（统计用）
    messages: list[dict] = []        # 完整对话历史
    current_understanding: dict = {} # A1当前理解的需求草案
    clarification_points: list[dict] = []  # A1识别的待澄清点
    human_confirmations: list[str] = []    # 人工已确认的部分
    last_updated: str = ""           # 最后更新时间
    
    def add_message(self, role: str, content: dict):
        """添加对话消息"""
        self.messages.append({
            "role": role,  # "human" | "ai"
            "content": content,
            "timestamp": datetime.now(timezone.utc).isoformat()
        })
        self.iterations += 1
        self.last_updated = datetime.now(timezone.utc).isoformat()
```

#### 核心逻辑：持续理解 + 反馈循环

```python
async def continuous_dialogue_loop(self, req_id: str, context_package: dict) -> dict:
    """持续对话模式主循环"""
    
    # 初始化对话状态
    state = ContinuousDialogueState()
    state.status = "active"
    
    # 提取初始需求
    initial_message = self._extract_message(context_package)
    state.add_message("human", {"text": initial_message, "type": "initial"})
    
    # 持续监听模式
    while state.status == "active":
        
        # ═══ Step 1: A1分析当前所有对话 ═══
        draft = await self._analyze_dialogue_history(
            messages=state.messages,
            req_id=req_id,
            knowledge_context=await self._fetch_knowledge_via_mcp(state)
        )
        state.current_understanding = draft
        
        # ═══ Step 2: 识别待澄清点（而非主动提问） ═══
        clarification_points = self._identify_unclear_points(draft)
        state.clarification_points = clarification_points
        
        # ═══ Step 3: 发布当前理解 + 待澄清点给人工 ═══
        await self._publish_understanding_snapshot(req_id, {
            "current_draft": draft,
            "clarification_points": clarification_points,
            "confidence_score": draft.get("confidence_score", 0),
            "iteration": state.iterations
        })
        
        # ═══ Step 4: 等待人工反馈（3种可能） ═══
        human_feedback = await self._wait_for_human_feedback(req_id)
        
        if human_feedback["action"] == "supplement":
            # 人工补充更多信息 → 继续循环
            state.add_message("human", {
                "text": human_feedback["content"],
                "type": "supplement",
                "addresses": human_feedback.get("addresses_points", [])  # 解决了哪些澄清点
            })
            continue
        
        elif human_feedback["action"] == "confirm":
            # 人工确认需求完善 → 进入下游
            state.status = "confirmed"
            state.human_confirmations = human_feedback.get("confirmed_sections", [])
            break
        
        elif human_feedback["action"] == "reject":
            # 人工拒绝/重置 → 记录并继续
            state.add_message("human", {
                "text": human_feedback.get("reason", "需求理解有误，请重新分析"),
                "type": "correction"
            })
            continue
    
    # ═══ Step 5: 最终发布确认后的草案 ═══
    state.status = "completed"
    final_draft = state.current_understanding
    final_draft["dialogue_state"] = state.dict()
    
    await self.report_artifact(req_id, "requirement_draft", final_draft)
    
    return {
        "status": "completed",
        "iterations": state.iterations,
        "requirement_draft": final_draft
    }
```

#### 待澄清点识别（非主动提问）

A1不再生成"问题"，而是识别"不确定的部分"：

```python
def _identify_unclear_points(self, draft: dict) -> list[dict]:
    """识别当前草案中的不确定点"""
    unclear_points = []
    
    # 1. 置信度低的字段
    if draft.get("confidence_score", 1.0) < 0.7:
        unclear_points.append({
            "category": "overall_confidence",
            "description": "需求描述较为简略，可能遗漏关键信息",
            "severity": "medium"
        })
    
    # 2. 关键字段缺失
    required_fields = {
        "domain": "业务领域未明确",
        "entities": "核心实体未识别",
        "acceptance_criteria": "验收标准缺失"
    }
    for field, desc in required_fields.items():
        if not draft.get(field):
            unclear_points.append({
                "category": f"missing_{field}",
                "description": desc,
                "severity": "high"
            })
    
    # 3. 歧义实体
    entities = draft.get("entities", {})
    if "user_role" not in entities and "用户" in draft.get("summary", ""):
        unclear_points.append({
            "category": "ambiguous_entity",
            "description": "用户角色不明确（管理员、普通用户、游客？）",
            "severity": "medium",
            "suggestions": ["请明确用户角色类型和权限范围"]
        })
    
    # 4. 技术栈缺失（复杂需求）
    if not draft.get("tech_stack_suggestion") and len(draft.get("acceptance_criteria", [])) > 5:
        unclear_points.append({
            "category": "missing_tech_stack",
            "description": "复杂需求建议明确技术选型",
            "severity": "low",
            "suggestions": ["可补充偏好的技术栈或团队现有技术栈"]
        })
    
    return unclear_points
```

```python
    finally:
        await nc.close()
        await websocket.close()
```

- LLM负责整合对话历史 + 知识库内容，生成理解

#### A1集成MCP的架构

```python
class A1RequirementIntakeV3(BaseAgentWorker):
    """v3.0 - 集成MCP知识库"""
    
    def __init__(self, nats_url: str = "nats://localhost:4222"):
        super().__init__("A1", "requirement_intake", nats_url)
        self.mcp_client = MCPClient("knowledge-base-mcp")
    
    async def _analyze_dialogue_history(
        self, 
        messages: list[dict], 
        req_id: str,
        knowledge_context: dict
    ) -> dict:
        """分析对话历史 + 知识增强"""
        
        # 构建对话历史prompt
        dialogue_text = self._format_dialogue_for_llm(messages)
        
        # 系统prompt：指导LLM使用MCP tools
        system_prompt = """你是需求分析专家。分析用户的需求对话，生成结构化需求草案。

**你可以使用的知识库工具**：
1. search_similar_requirements - 搜索历史相似需求
2. get_tech_stack_recommendations - 获取技术栈推荐
3. get_domain_risks - 获取领域常见风险
4. get_cost_baseline - 获取成本基线

**工作流程**：
1. 理解对话内容，提取核心需求
2. 识别业务领域（domain）
3. 调用MCP工具获取知识（相似案例、技术栈、风险点）
4. 整合对话 + 知识，输出结构化草案

**输出格式（JSON）**：
{
  "title": "需求标题",
  "domain": "业务领域",
  "summary": "需求概要",
  "confidence_score": 0.85,
  "entities": {...},
  "acceptance_criteria": [...],
  "tech_stack_suggestion": {
    "backend": "...",
    "frontend": "...",
    "rationale": "基于知识库推荐 + 历史案例"
  },
  "risk_points": ["风险1（来源：历史案例REQ-001）"],
  "knowledge_sources": ["REQ-001", "REQ-015"]
}
"""
        
        # 调用LLM with MCP tools
        draft = await self.call_llm_with_mcp_tools(
            messages=[
                {"role": "system", "content": system_prompt},
                {"role": "user", "content": dialogue_text}
            ],
            available_tools=self.mcp_client.get_tools(),
            task_type="requirement_analysis_with_knowledge",
            req_id=req_id,
            temperature=0.3,
            max_tokens=3000
        )
        
        return draft
    
    async def call_llm_with_mcp_tools(
        self, 
        messages: list[dict],
        available_tools: list[dict],
        task_type: str,
        req_id: str,
        **kwargs
    ) -> dict:
        """调用LLM，支持MCP tool use"""
        
        # 调用LLM Provider with tool definitions
        response = await self.llm_provider.call(
            messages=messages,
            tools=available_tools,
            tool_choice="auto",
            **kwargs
        )
        
        # 处理tool calls（如果LLM选择使用工具）
        if response.get("tool_calls"):
            tool_results = []
            
            for tool_call in response["tool_calls"]:
                tool_name = tool_call["function"]["name"]
                tool_args = json.loads(tool_call["function"]["arguments"])
                
                # 调用MCP工具
                result = await self.mcp_client.call_tool(tool_name, tool_args)
                tool_results.append({
                    "tool_call_id": tool_call["id"],
                    "role": "tool",
                    "name": tool_name,
                    "content": json.dumps(result)
                })
            
            # 将工具结果返回给LLM继续处理
            messages.append(response["message"])
            messages.extend(tool_results)
            
            # 递归调用LLM处理工具结果
            final_response = await self.llm_provider.call(
                messages=messages,
                tools=available_tools,
                **kwargs
            )
            
            return json.loads(final_response["message"]["content"])
        
        else:
            # LLM直接返回结果（未使用工具）
            return json.loads(response["message"]["content"])
```

#### MCP Client实现

```python
class MCPClient:
    """MCP服务客户端"""
    
    def __init__(self, service_name: str):
        self.service_name = service_name
        self.base_url = os.getenv(f"MCP_{service_name.upper().replace('-', '_')}_URL", 
                                   f"http://localhost:8100/mcp/{service_name}")
        self.tools_cache = None
    
    async def get_tools(self) -> list[dict]:
        """获取MCP服务提供的工具列表"""
        if self.tools_cache:
            return self.tools_cache
        
        async with httpx.AsyncClient() as client:
            resp = await client.get(f"{self.base_url}/tools")
            resp.raise_for_status()
            self.tools_cache = resp.json()["tools"]
            return self.tools_cache
    
    async def call_tool(self, tool_name: str, arguments: dict) -> dict:
        """调用MCP工具"""
        async with httpx.AsyncClient(timeout=10.0) as client:
            resp = await client.post(
                f"{self.base_url}/tools/{tool_name}",
                json=arguments
            )
            resp.raise_for_status()
            return resp.json()
```

#### MCP vs A2对比

| 维度 | 旧方案（A2 Agent） | 新方案（MCP） |
|------|------------------|--------------|
| **调用方式** | NATS request-reply | HTTP/SSE MCP协议 |
| **知识整合** | A2返回结构化数据，A1合并 | LLM直接调用工具，自主整合 |
| **灵活性** | A2固定返回格式 | LLM按需选择调用哪些工具 |
| **容错性** | A2超时→降级 | 单个工具失败不影响其他 |
| **可观测性** | NATS消息追踪 | MCP工具调用日志 |
| **复杂度** | 需维护A2 Agent | 仅需轻量级MCP服务 |

```python
async def search_similar_requirements(query: SimilarRequirementsQuery):
    """搜索相似需求（使用pgvector）"""
    conn = await asyncpg.connect(os.getenv("DATABASE_URL"))
    
    # 使用向量相似度搜索
    sql = """
        SELECT id, title, spec->>'domain' as domain, 
               spec->>'tech_stack_suggestion' as tech_stack,
               1 - (embedding <=> $1::vector) as similarity
        FROM requirements
        WHERE ($2::text IS NULL OR spec->>'domain' = $2)
          AND (1 - (embedding <=> $1::vector)) >= $3
        ORDER BY similarity DESC
        LIMIT $4
    """
    
    # 将query转换为向量（调用embedding模型）
    query_embedding = await get_embedding(query.query)
    
    rows = await conn.fetch(
        sql, 
        query_embedding, 
        query.domain, 
        query.similarity_threshold,
        query.top_k
    )
    
    await conn.close()
    
    return {
        "results": [
            {
                "req_id": str(row["id"]),
                "title": row["title"],
                "domain": row["domain"],
                "similarity": float(row["similarity"]),
                "tech_stack": json.loads(row["tech_stack"]) if row["tech_stack"] else {}
            }
            for row in rows
        ]
    }
```

#### 插件接口标准

```python
from abc import ABC, abstractmethod

class A1Skill(ABC):
    skill_id: str
    skill_name: str
    version: str = "1.0"
    
    @abstractmethod
    def should_apply(self, draft: dict, context: dict) -> bool:
        """判断是否应用此Skill（预检查）"""
        pass
    
    @abstractmethod
    async def execute(self, draft: dict, context: dict) -> dict:
        """执行Skill，返回增强结果"""
        pass
    
    def get_metadata(self) -> dict:
        return {
            "skill_id": self.skill_id,
            "skill_name": self.skill_name,
            "version": self.version,
        }
```

```python
class RiskAssessmentSkill(A1Skill):
    skill_id = "risk_assessment"
    skill_name = "风险评估专家"
    
    def should_apply(self, draft: dict, context: dict) -> bool:
        # 需求优先级>=P1 或 领域为支付/订单时才评估风险
        return draft.get("priority_suggestion") in ["P0", "P1"] or \
               draft.get("domain") in ["payment", "order_management"]
    
    async def execute(self, draft: dict, context: dict) -> dict:
        """4维度风险评估：技术复杂度、业务复杂度、依赖风险、时间压力"""
        risk_score = await self._call_risk_llm(draft)
        
        return {
            "risk_assessment": {
                "overall_score": risk_score["overall"],  # 0-100
                "dimensions": {
                    "technical_complexity": risk_score["tech"],      # 技术复杂度
                    "business_complexity": risk_score["business"],   # 业务复杂度
                    "dependency_risk": risk_score["dependency"],     # 依赖风险
                    "time_pressure": risk_score["time"],             # 时间压力
                },
                "risk_level": "high" if risk_score["overall"] > 70 else ("medium" if risk_score["overall"] > 40 else "low"),
                "recommendations": risk_score["recommendations"]
            }
        }
    
    async def _call_risk_llm(self, draft: dict) -> dict:
```

```python
class SkillsOrchestrator:
    def __init__(self, config_path: str = ".ai-native/a1-skills.yaml"):
        self.skills: list[A1Skill] = []
        self._load_builtin_skills()
        self._load_external_skills(config_path)
    
    def _load_builtin_skills(self):
        self.skills.extend([
            RiskAssessmentSkill(),
            CostEstimationSkill(),
            TechStackSkill(),
        ])
    
    def _load_external_skills(self, config_path: str):
        if not os.path.exists(config_path):
            return
        
        with open(config_path) as f:
            config = yaml.safe_load(f)
        
        ext_dir = config.get("external_skills_dir", "./skills/a1")
        if not os.path.exists(ext_dir):
            return
        
        # 动态加载Python模块
        for filename in os.listdir(ext_dir):
            if filename.endswith(".py"):
                module = importlib.import_module(f"skills.a1.{filename[:-3]}")
                for attr in dir(module):
                    cls = getattr(module, attr)
                    if isinstance(cls, type) and issubclass(cls, A1Skill) and cls != A1Skill:
                        self.skills.append(cls())
    
    async def apply_skills(self, draft: dict, context: dict) -> dict:
        """串行应用所有适用的Skills"""
        for skill in sorted(self.skills, key=lambda s: getattr(s, 'priority', 999)):
            if skill.should_apply(draft, context):
                logger.info(f"[A1] Applying skill: {skill.skill_name}")
                try:
                    result = await asyncio.wait_for(
                        skill.execute(draft, context),
                        timeout=skill.timeout_seconds
                    )
                    draft.update(result)
                except asyncio.TimeoutError:
                    logger.warning(f"[A1] Skill {skill.skill_id} timeout")
                except Exception as e:
                    logger.error(f"[A1] Skill {skill.skill_id} error: {e}")
        
        return draft
```

```python
async def execute_v3(self, req_id: str, context_package: dict) -> dict:
    """A1 v3.0 - 持续对话模式"""
    
    # ═══ 初始化 ═══
    state = ContinuousDialogueState(status="active")
    initial_message = self._extract_message(context_package)
    state.add_message("human", {"text": initial_message, "type": "initial"})
    
    # 初始化MCP客户端
    mcp_client = MCPClient("knowledge-base-mcp")
    
    # ═══ 持续对话循环 ═══
    while state.status == "active":
        iteration = state.iterations
        
        # ─── Step 1: LLM分析对话历史 + MCP知识增强 ─────
        await self.report_status(req_id, "running", 
            f"迭代 {iteration}: 分析对话并整合知识库")
        
        draft = await self._analyze_dialogue_with_mcp(
            messages=state.messages,
            req_id=req_id,
            mcp_client=mcp_client
        )
        
        if not draft:
            # LLM失败，降级到关键词模式
            logger.warning(f"[A1] LLM analysis failed, fallback to keywords")
            draft = self._fallback_analysis(state.messages[-1]["content"]["text"])
        
        state.current_understanding = draft
        
        # ─── Step 2: 识别待澄清点（不是问题，是不确定的部分） ─────
        clarification_points = self._identify_unclear_points(draft)
        state.clarification_points = clarification_points
        
        # ─── Step 3: 发布理解快照到前端（WebSocket推送） ─────
        await self._publish_understanding_snapshot(req_id, {
            "iteration": iteration,
            "current_draft": {
                "title": draft.get("title"),
                "domain": draft.get("domain"),
                "summary": draft.get("summary"),
                "confidence_score": draft.get("confidence_score", 0),
                "acceptance_criteria": draft.get("acceptance_criteria", []),
                "tech_stack_suggestion": draft.get("tech_stack_suggestion"),
            },
            "clarification_points": clarification_points,
            "knowledge_sources": draft.get("knowledge_sources", []),
            "timestamp": datetime.now(timezone.utc).isoformat()
        })
        
        # ─── Step 4: 等待人工反馈（无超时，持续等待） ─────
        await self.report_status(req_id, "waiting", 
            f"等待人工反馈（{len(clarification_points)}个待澄清点）")
        
        feedback = await self._wait_for_human_feedback(req_id)
        
        # ─── Step 5: 处理反馈 ─────
        if feedback["action"] == "supplement":
            # 人工补充信息 → 继续循环
            state.add_message("human", {
                "text": feedback["content"],
                "type": "supplement",
                "addresses_points": feedback.get("addresses_points", [])
            })
            
            logger.info(f"[A1] req={req_id} received supplement, iteration {iteration+1}")
            continue
        
        elif feedback["action"] == "confirm":
            # 人工确认需求完善 → 结束循环
            state.status = "confirmed"
            state.human_confirmations = feedback.get("confirmed_sections", ["all"])
            
            logger.info(f"[A1] req={req_id} confirmed after {iteration} iterations")
            break
        
        elif feedback["action"] == "reject":
            # 人工纠正理解 → 记录并继续
            state.add_message("human", {
                "text": feedback.get("reason", "理解有误，请重新分析"),
                "type": "correction"
            })
            
            logger.warning(f"[A1] req={req_id} rejected, re-analyzing")
            continue
    
    # ═══ Step 6: 确认后Skills增强 ═══
    await self.report_status(req_id, "running", "应用Skills增强")
    orchestrator = SkillsOrchestrator()
    final_draft = await orchestrator.apply_skills(state.current_understanding, {
        "req_id": req_id,
        "iterations": state.iterations
    })
    
    # ═══ Step 7: 最终发布 ═══
    state.status = "completed"
    final_draft["dialogue_state"] = state.dict()
    
    await self.report_artifact(req_id, "requirement_draft", final_draft)
    
    return {
        "status": "completed",
        "iterations": state.iterations,
        "requirement_draft": final_draft,
        "source": "llm_mcp_v3"
    }


async def _analyze_dialogue_with_mcp(
    self, 
    messages: list[dict], 
    req_id: str,
    mcp_client: MCPClient
) -> dict:
    """核心：LLM分析对话 + 调用MCP工具获取知识"""
    
    # 格式化对话历史
    dialogue_text = "\n\n".join([
        f"[{msg['role'].upper()}] {msg['content'].get('text', '')}"
        for msg in messages
    ])
    
    # 系统prompt
    system_prompt = """你是需求分析专家。分析用户的需求对话，生成结构化需求草案。

**可用的知识库工具（MCP）**：
1. search_similar_requirements(query, domain, top_k) - 搜索历史相似需求
2. get_tech_stack_recommendations(domain, complexity) - 获取技术栈推荐
3. get_domain_risks(domain) - 获取领域常见风险
4. get_cost_baseline(domain, features_count) - 获取成本基线

**工作流程**：
1. 理解完整对话，识别业务领域
2. 调用MCP工具获取历史知识
3. 整合对话内容 + 知识库推荐，生成草案
4. 评估置信度（0-1），低于0.7表示信息不足

**输出JSON格式**：
{
  "title": "需求标题",
  "domain": "业务领域",
  "summary": "需求概要",
  "confidence_score": 0.85,
  "entities": {...},
  "acceptance_criteria": [...],
  "tech_stack_suggestion": {
    "backend": "推荐方案",
    "rationale": "基于知识库的X个案例推荐"
  },
  "risk_points": ["风险1（来源：历史案例REQ-001）"],
  "knowledge_sources": ["REQ-001"]
}
"""
    
    # 调用LLM with MCP tools
    return await self.call_llm_with_mcp_tools(
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": dialogue_text}
        ],
        available_tools=mcp_client.get_tools(),
        mcp_client=mcp_client,
        task_type="requirement_analysis_with_knowledge",
        req_id=req_id,
        temperature=0.3,
        max_tokens=3000
    )


async def _wait_for_human_feedback(self, req_id: str) -> dict:
    """等待人工反馈（无超时，持续等待）"""
    
    feedback_subject = f"requirement.feedback.{req_id}"
    feedback_future = asyncio.Future()
    
    async def feedback_handler(msg):
        data = json.loads(msg.data.decode())
        if not feedback_future.done():
            feedback_future.set_result(data)
    
    await self.nc.subscribe(feedback_subject, cb=feedback_handler)
    
    try:
        # 无超时，持续等待
        feedback = await feedback_future
        return feedback
    finally:
        await self.nc.unsubscribe(feedback_subject)


async def _publish_understanding_snapshot(self, req_id: str, snapshot: dict):
    """发布A1当前理解快照到前端"""
    
    # 发布到NATS供WebSocket转发
    await self.nc.publish(
        f"requirement.understanding.{req_id}",
        json.dumps(snapshot, ensure_ascii=False).encode()
    )
    
    # 同时记录到数据库（审计用）
    await self.report_artifact(req_id, "understanding_snapshot", snapshot)
```

### Skills沙箱机制

```python
class SkillSandbox:
    """Skills执行沙箱，防止恶意插件影响系统"""
    
    def __init__(self, timeout: int = 30, max_memory_mb: int = 512):
        self.timeout = timeout
        self.max_memory_mb = max_memory_mb
    
    async def execute_skill(self, skill: A1Skill, draft: dict, context: dict) -> dict:
        """在沙箱中执行Skill"""
        
        # 1. 权限检查
        if not self._check_permissions(skill):
            raise PermissionError(f"Skill {skill.skill_id} lacks required permissions")
        
        # 2. 超时控制
        try:
            result = await asyncio.wait_for(
                skill.execute(draft, context),
                timeout=self.timeout
            )
        except asyncio.TimeoutError:
            logger.error(f"Skill {skill.skill_id} timeout after {self.timeout}s")
            return {}
        
        # 3. 输出校验
        if not isinstance(result, dict):
            logger.error(f"Skill {skill.skill_id} returned invalid type: {type(result)}")
            return {}
        
        return result
    
    def _check_permissions(self, skill: A1Skill) -> bool:
        """检查Skill权限（禁止文件写入、网络访问等）"""
        metadata = skill.get_metadata()
        required_perms = metadata.get("required_permissions", [])
        
        # 黑名单检查
        forbidden = ["file_write", "network_external", "subprocess"]
        if any(p in forbidden for p in required_perms):
            return False
        
        return True
```

### 降级策略

**三级降级保障**：

1. **MCP工具级降级**：单个MCP工具失败 → 跳过该工具，LLM基于已有信息分析
2. **Skill级降级**：单个Skill失败 → 跳过该Skill，继续执行其他Skills
3. **v3.0级降级**：持续对话/MCP全部失败 → 回退到v1.0单次LLM模式

```python
async def execute_with_fallback(self, req_id: str, context_package: dict) -> dict:
    """带降级保障的执行入口"""

        # 降级到v1.0单次LLM
        return await self.execute_v1(req_id, context_package)
```

### 关键设计决策

**决策1：为什么改为持续对话？**
- 问题：固定3轮问答限制了需求澄清深度，复杂需求3轮不够
- 方案：改为持续对话，人工决定何时完善
- 收益：准确度提升、人工体验更自然

**决策2：为什么用MCP替代A2？**
- 问题：A2 Agent增加系统复杂度，NATS调用链路长
- 方案：MCP协议标准化知识访问，LLM直接调用工具
- 收益：架构简化、LLM自主决策、容错性提升

**决策3：Skills何时执行？**
- 问题：每次迭代都执行Skills浪费资源
- 方案：仅在人工确认后执行（基础信息已完善）
- 收益：性能优化、避免基于不完整信息评估

## 总结与展望

### 核心价值主张

A1长期演进围绕**三大核心能力**：

1. **多轮对话**（Dialogue） - 主动澄清模糊需求
   - 从"被动解析"到"主动引导"
   - 置信度驱动的智能决策
   - 最多3轮澄清，避免过度交互

2. **知识增强**（Knowledge） - 历史需求复用
   - 接入A2知识库，召回top-5相似案例
   - 技术栈投票推荐，降低选型风险
   - 风险点聚合，提前识别高频问题

3. **插件化**（Extensibility） - 动态扩展能力
   - Skills生态：风险评估、成本估算、领域检测
   - 配置化启用/禁用
   - 沙箱隔离，保障系统稳定性

### 从"需求解析器"到"智能需求分析专家"

| 维度 | v1.0 解析器 | v3.0 专家系统 |
|------|------------|--------------|
| **交互模式** | 单次输入→输出 | 多轮对话式引导 |
| **知识来源** | 仅通用LLM | LLM + 历史库 + Skills |
| **准确度** | 70%（依赖用户表达清晰度） | 90%+（主动消歧） |
| **适应性** | 固化流程 | 自适应策略 |
| **扩展性** | 硬编码逻辑 | 可插拔Skills |

### 未来演进方向（v4.0+）

**短期优化**（3-6个月）：
- 引入多模态输入（用户上传原型图、流程图）
- 增强LLM上下文（注入公司技术标准、编码规范）
- 支持需求模板库（常见场景快速启动）

**长期探索**（1年+）：
- **Agent自主学习**：通过Gate反馈和实际交付质量微调LLM
- **跨需求关联分析**：识别需求间的依赖和冲突
- **成本-收益优化器**：自动建议需求优先级排序
- **可视化需求编辑器**：交互式调整acceptance_criteria和tech_stack

### 关键成功因素

1. **A2知识库质量**：历史需求数据需≥200条才能有效训练
2. **用户反馈闭环**：Gate 0人工审核结果需回流A1改进
3. **Skills生态建设**：鼓励团队贡献领域专用Skills
4. **监控体系完善**：实时感知v3.0降级情况，快速止血

### 结语

A1 v3.0的核心价值不在于技术复杂度，而在于**降低需求分析的摩擦成本**：
- 用户无需完美表达 → A1主动澄清
- 研发无需重复踩坑 → A2提供历史经验
- 系统可按需扩展 → Skills插件化

从"10分钟写需求文档"到"2分钟对话生成高质量需求"，这是A1演进的终极目标。
