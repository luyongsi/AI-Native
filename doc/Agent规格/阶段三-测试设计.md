## 一、测试策略总览

### 1.1 三层覆盖

| 测试层级 | 覆盖范围 | 执行频率 | 工具 |
|---------|---------|---------|------|
| **单元测试** | A6/A7/A8 各自的核心逻辑（LLM prompt 解析、DAG 验证、静态检查算法、fallback 逻辑） | 每次 git push | pytest |
| **集成测试** | NATS 事件收发、PostgreSQL 读写、LLM API 调用（mock）、Orchestrator 编排 | 每日构建 | pytest + pytest-asyncio + unittest.mock |
| **端到端测试** | 完整流程：Gate1 pass → A6+A7 并行 → A8 评审 → Gate2 pass/reject | 里程碑检查 | 手动/自动化脚本 |

### 1.2 Mock 策略

| 组件 | Mock 方式 | 原因 |
|------|----------|------|
| DeepSeek API | `unittest.mock.patch` / `httpx.MockTransport` | 不依赖外部 LLM 服务 |
| NATS JetStream | pytest fixture 提供 mock JS context | 不依赖 NATS 服务 |
| PostgreSQL | 使用测试数据库 + 事务回滚 | 验证真实 SQL |
| MC Backend API | `httpx.MockTransport` | 验证 HTTP 调用格式 |

### 1.3 测试数据准备

- **Spec 模板**：创建 `tests/fixtures/spec_complete.json`（含 5 个模块 + 8 个 API 端点 + 4 个 ERD 实体）
- **空 Spec 模板**：`tests/fixtures/spec_empty.json`（仅标题）
- **DAG 模板**：`tests/fixtures/dag_valid.json`（10 个节点 + 12 条边）
- **DAG 含循环模板**：`tests/fixtures/dag_cyclic.json`
- **DAG 分层违规模板**：`tests/fixtures/dag_layer_violation.json`

## 七、集成测试场景

### IT-001: 端到端 — Gate1 pass → Gate2 pass

```
Gate1 pass
  → Orchestrator: phase='tech_prep', status='decomposing'
  → context.ready.A6 (含 spec_package + revision_context)
  → context.ready.A7 (含 spec_package + dag_preview)
  → A6 生成 DAG (8 nodes, 10 edges) → agent.result.A6
  → A7 生成 test cases (16 cases) → agent.result.A7
  → Orchestrator GATHER → context.ready.A8
  → A8 评审 (score=82, verdict=pass) → agent.result.A8
  → Orchestrator → context.ready.gate2
  → Gate2 pass → phase='development' → context.ready.A9
```

**验证点**:
- requirements.tech_prep_status 状态流转: decomposing → decomposed/test_ready → reviewing → tech_prep_completed
- task_dags 表含 1 行（version=1）
- test_assets 表含 1 行（version=1）
- agent_results 含 A6/A7/A8 (cycle=0, status='completed')
- approvals 表含 gate_level=2, decision='pass'
- event_log 含全部收发记录

### IT-002: 端到端 — Gate2 reject → 重新流转

```
Gate1 pass → A6+A7 已完成 → A8 评审 → Gate2 reject
  → tech_prep_revision_count: 0 → 1
  → context.ready.A6 (revision_context.is_revision=true)
  → context.ready.A7 (revision_context.is_revision=true)
  → A6 修订 DAG (upsert agent_results, task_dags version=2)
  → A7 修订 test cases (new test_assets version=2)
  → A8 重新评审
  → Gate2 pass → phase='development'
```

**验证点**:
- cycle 始终为 0（不递增）
- tech_prep_revision_count = 1
- task_dags version = 2
- test_assets version = 2
- agent_results A6/A7 同 cycle 覆盖旧记录

### IT-003: 降级链路 — A6 超时 → 空 DAG 流转 Gate2

```
Gate1 pass → A6 超时 → a6_missing=true
  → A7 正常完成
  → context.ready.gate2 (a6_output.a6_missing=true)
  → Gate2 审批人看到空 DAG 提示
  → Gate2 reject → 触发重新尝试 A6
```

### IT-004: 并发安全性 — 同一 req 多次调度

```
Gate1 pass → A6+A7 并行
  → Orchestrator 收到重复的 agent.result.A6（双发）
  → GATHER 逻辑仅触发一次（幂等）
  → 验证 agent_results UPSERT 去重
```
