NEW API ENDPOINTS
=================
POST /api/tests/{req_id}/cases/{case_id}/validate
  - Triggers A7 validation
  - Returns: { status, errors, coverage, suggestions }
  - NATS event: test.validate → A7 agent → test.validated

POST /api/tests/{req_id}/cases/augment
  - Triggers A11 augmentation
  - Returns: { status, additional_count, message }
  - NATS event: test.augment_request → A11 agent → test.augmented

DATABASE CHANGES
================
Added to test_cases table:
  - validation_status VARCHAR(20) [pending/passed/failed]
  - validation_errors JSONB [error array]
  - coverage_info JSONB [coverage metrics]
  - source VARCHAR(50) [manual/a7_generated/a11_generated]
  - augmentation_suggestions JSONB [suggestions array]

New indexes:
  - idx_test_cases_validation_status
  - idx_test_cases_source
  - idx_test_cases_req_validation

NATS EVENTS
===========
test.validate (API → A7)
  { req_id, test_case, event_type }

test.validated (A7 → DB/Frontend)
  { req_id, case_id, status, errors, coverage }

test.augment_request (API → A11)
  { req_id, analysis_type }

test.augmented (A11 → DB/Frontend)
  { req_id, additional_count, metrics }

FRONTEND COMPONENT
==================
Usage:
  <TestCaseEditor reqId="uuid" onClose={handler} />

Features:
  - Create/edit test cases
  - View list with filtering
  - Modal editor with step builder
  - Real-time ActivityStream sidebar
  - Priority selector (P0/P1/P2/P3)
  - Source indicator (manual/AI-generated)
  - Validation status display
  - Coverage info visualization
  - Save/Validate/Augment buttons

STATE MACHINE
=============
Test Case States:
  - pending: Created, not validated
  - passed: Validation succeeded
  - failed: Validation failed
  - validated: After A7 validation
  - augmented: After A11 augmentation

VALIDATION RULES
================
Syntax Checks:
  ✓ Required fields (title, steps, priority)
  ✓ Valid priority (P0/P1/P2/P3)
  ✓ Correct step structure
  ✓ Title < 200 chars

Logic Checks:
  ✓ Sequential step numbers
  ✓ Valid preconditions format
  ✓ Reasonable description length
  ✓ Valid tags format

AUGMENTATION STRATEGY
====================
1. Fetch existing test cases
2. Analyze coverage gaps
3. Identify uncovered paths
4. Generate tests (Critic mode)
5. Create mutation survivor tests
6. Save as a11_generated
7. Set priority by coverage gap severity

INTEGRATION FLOW
================
Validation:
  User edits → Click Validate → API publishes test.validate
  → A7 processes → API publishes test.validated
  → Frontend shows results in ActivityStream

Augmentation:
  User clicks Augment → API publishes test.augment_request
  → A11 processes → A11 publishes test.augmented
  → Frontend refreshes with new tests
